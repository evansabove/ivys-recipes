# Recipe Creation Rules

- **NEVER invent or generate recipe content** (ingredients, quantities, steps, tips, etc.) when asked to create a new recipe.
- The user always has a specific recipe in mind. When asked to create one, **ask the user to provide the recipe details** before creating the file.
- You may create the branch and an empty/skeleton recipe file with placeholder frontmatter, but leave all content fields blank or clearly marked as `TODO` until the user supplies the details.
- Only use the existing recipes (e.g. `apple-crumble.md`) as a structural reference — not as a source of content to copy or adapt.
